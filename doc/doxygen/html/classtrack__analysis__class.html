Detailed Description

Estimates stop times for a vehicle track and writes these to a shapefile.

The class tries to estimates stops that are important. It weeds out stops that are too short since these are probably owing to normal traffic. It also tries to account for registration errors that may make it appear that a stopped vehicle is moving.

Attributes are copied from a specified file and transfered to the output shapefile. The aim is to have shapefile attributes in the format needed by the analysts and to fill in as many of these attributes as can be done automatically.

The first step in the estimation algorithm is to find all intervals from one user-input point to the next where the speed of the vehicle is below the threshold speed. The second step is to merge any of these intervals that share end points. The third step is to cull out any intervals that are shorter than the specified time threshold.
