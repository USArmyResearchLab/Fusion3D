Get the texture image – default option when available, otherwise use 'mrg' or 'int' files Typically uses MrSID helper class to decode (downsampled for medium and low res) data but may also use .tif helper class.

Parameters
north_cenY- North-coordinate of center of tile in m
east_cenX- East-coordinate of center of tile in m
height_tileY-size of tile in m
width_tileX-size of tile in m
dataLocation where texture data is put
res_flag1=hi-res, 2=med-res, 3=low-res
i_groupIndex to multiple parallel instances (each handles all images) – last instance is lo-res, next-to-last is med-res
Returns
0 if no intersection, 1 if intersection with rgb or if totally off the map (just fill with ibrt_offmap), 2 if intersection with all textures gray

Only adds false color when all the texture images are grayscale.

rainbow_rminElevation corresponding to minimum color
rainbow_rmaxElevation corresponding to maximum color

Modify a1-DEM texture data array to indicate change – reds for additions, blues for subtractions.

Called once per hi-res tile – hi-res assumed. Uses an elevation-difference DEM-like GeoTiff file whose name is derived from the a1-DEM filename using BuckEye conventions.

Parameters
north_cenY/North-coordinate of center of tile in m
east_cenX/East-coordinate of center of tile in m
height_tileY-size of tile in m
width_tileX-size of tile in m
dataLocation of texture data, input and output
