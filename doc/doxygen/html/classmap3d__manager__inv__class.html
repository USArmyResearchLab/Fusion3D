Detailed Description

Adds OpenInventor functionality to map3d_manager_class.

Adds capability to draw the 3-D elevation maps that the class constructs.

Forced retile: When certain display parameters change, certain tiles need to be regenerated as follows:

Toggling vis of first hit surface: Only the medium and hi tiles and only when draping is on (otherwise, nothing needs regen)
Toggling draping; Only the medium and hi tiles (low tiles are already draped)
Toggling highlighting: Only the medium and hi tiles (low tiles are never highlighted)
LOS: Only the medium and hi tiles

Manages multithreading: The default is to set the multithreading automatically based on the number of processors and the graphics library used. With the VSG library, stronger multithreading capability allows tiles to be built in a worker thread, so the default is to make all tiles in the worker thread. This can be overridden by the user. For the Coin3D library, weaker multithreading doesnt allow tiles to be built in the worker thread, so all tiles must be built in the main thread. However, low-res tiles can be built by the main thread in the background (using an IdleSensor that only triggers when no other actions are being processed) and this is turned on by default for Coin3D.

MrSID decoding for the hi-res tiles is compute intensive, so read-decode (no OpenInventor) is done in separate threads if there are enough processors. If there are 4 processors, then 2 helper threads are used. If 2 processor, 1 helper thread is used. If only 1 processor, no MrSID threads are used.

Threads are initialized when the viewer is started and left unchanged until the viewer is killed.
