CONFIGURATION FILE 'DOXYFILE'

Doxygen runs with a configuration file 'Doxyfile' that we have generated. This file has a lot of tag/value combinations that define how Doxygen reads and parses files. The most interesting tags are given below -- descriptions of the tags are in the file: 

	INPUT                  = ../../src ../../include ../../apps ../../ui
	EXAMPLE_PATH           = ./AdditionalDocFiles
	PROJECT_NAME           = Fusion3D
	PROJECT_NUMBER         = "Version 6.04"
	OUTPUT_DIRECTORY       =

RUNNING DOXYGEN USING DOXYWIZARD GUI


1.  Run Doxywizard and specify the working directory
	This should be the base doxygen directory 'doxygen'

2.  Open the config file
	File -> Open -> select file doxygen/Doxyfile

3.  Click the 'Run' tab and click 'Run Doxygen'

DIRECTIVES WITHIN CODE

Exclude Code from Doxygen


/// @cond		Exclude from Doxygen
	Excluded code
/// @endcond	End Doxygen exclusion

General Doc for .h File

The Doxygen code must immediately preceed the beginning of the class definition. For example 

/**
This class presents the Overlay Transparancy Dialog to the user.

This class allows the user to enter the overlay transparency parameters to
the user. This includes the opacity percent.

*/
class dialog_adjust_overlay_transparency : public CDialog

Variable Doc for .h File

You can put doc after each variable, but must indicate that doc is for preceding variable (otherwise, it will associate it with the variable on the next line). For example 

 protected:
   int n_max;					///< Max no of locs that can be read/stored

Gotchas


Visual Studio does not put ';' after following which generates Doxygen error
	DECLARE_DYNAMIC(dialog_route_class);
	DECLARE_MESSAGE_MAP();
